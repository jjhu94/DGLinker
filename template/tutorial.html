0.Quick start

To run a job please follow the steps below (Figure 1):
1. Go to the homepage (https://dglinker.rosalind.kcl.ac.uk);
2. (Optional but recommended) Enter your email address if you want to receive job notifications and details, and a link to the results page via email;
3. Choose the input type you want to use to predict candidate genes;
4. Load the example;
5. Click “Predict”.

1.How to run a basic job

To run a job please follow the steps below (Figure 2):
1. Go to the homepage (https://dglinker.rosalind.kcl.ac.uk);
2. (Optional but recommended) Enter your email address if you want to receive job notifications and details, and a link to the results page via email;
3. Enter the phenotype(s) you want to predict candidate genes for;
4. Click “Predict”.

Figure 2



2.Input details

For simplified imformation, you can put the cursor on the “?” logo next to the corresponding title (Figure 3).

Figure 3


2.1 Email address

Providing an email address is optional although strongly recommended (Figure 1). If provided, the webserver will send the following: 1) a confirmation email, when the job is submitted, with the job ID and a link to the results page, where the user will be able to visualise the results when ready; 2) when the job is completed, a completion email to alert the user that either the results are now available or, in case of error, that the job was not completed successfully.

2.2 Input options

2.2.1 Select phenotype(s)

This is the default input option (Figure 4). DGLinker uses our current knowledge of the genes that are associated with the target phenotype to predict new candidate genes. When one or more phenotypes are selected, DGLinker uses all genes associated with them on DisGeNet v7.0 (https://www.disgenet.org) as input known genes.

Figure 4


If “Retrieve related phenotypes” is selected, phenotypes matching the input keywords will be automatically selected (Figure 5).

Figure 5




2.2.2 Select phenotype(s) associated genes

If the user selects this option, they must provide both a list of genes and one or more phenotypes that the genes are associated with (Figure 6). These will be used to train the predictive model instead of the DisGeNet associations. Although DisGeNet is a comprehensive and curated database of gene-phenotype relationships, it is not unusual for researchers to have their own manually curated lists of phenotype-associated genes. These might reflect, for example, specific required supporting evidence of association, novel/not-yet established associations, the result of their own studies, or disease subtypes.

Figure 6




2.2.3 Select genes

If the user selects this option, they must provide a list of genes (Figure 7). Using this option assumes that the phenotype(s) the provided genes are associated with, is(are) not known to be the same or strongly related (e.g. a subtype) with any present in DisGeNet.

Figure 7




2.3 Available databases

By default, one gene-disease (DisGeNet), protein-protein interaction (IntAct) and the gene ontology (2020-11-17 release) datasets are selected (Figure 8). The user can select additional databases or remove any of the default ones by ticking the corresponding boxes. At least one database must be selected to start a job.

Figure 8




2.4 Upload database(s)

Users can upload their own dataset(s) of gene-phenotype or gene-gene relationships using the “Upload database(s)” tool (Figure 9a). These have to be in comma delimited csv format and include at least one “Gene” column. The HGNC nomenclature (https://www.genenames.org) must be used. If more than one database is to be uploaded, these must be selected and uploaded together (Figure 9b). Currently, there is a limit of 100Mb to upload datasets.

Figure 9a

3. Retrieve a job

This section of the homepage allows the results of a job to be retrieved by using the job ID that was generated when the job was submitted (Figure 10). The Job ID is displayed in the waiting page after submission and emailed to the user if an email address was provided.

Figure 10



4. Results page

After submitting a job, results should be available within minutes. However, this can vary substantially as the time needed to complete a job depends on the number and size of the databases selected, and the number of input genes. Do not worry if the server takes longer than expected. An error email is sent if the job fails as long as an email address was provided.
The results page is divided into the following 5 main tabs:

4.1 Job summary

This tab displays a table with the details of the job submission and number of predicted genes (Figure 11).

Figure 11



4.2 Results table

This tab displays an interactive table that contains the list of predicted and known genes, their prediction score and a number of links to external databases that contain useful information about the genes (Figure 12).

Figure 12



4.3 Gene networks

This tab allows the user to visualize the interaction networks of the top 50 genes (according to their score) from the knowledge graph used for the prediction (Figure 13a). By clicking on the individual genes, an interactive network visualization is displayed in the right panel of the tab. This individual gene network will include all terms the selected gene interacted directly with (1st degree neighbours), and all genes, among the top 50, that interacted with the selected gene neighbours.

Figure 13a


In the network graph, circular nodes represent genes and boxes represent non-gene nodes (e.g. GOterms). Different colours identify different classes of non-gene nodes (e.g. proteins, diseases, GOterms, etc), or distinguish between the central gene node (the selected gene we are visualising the network of), known genes and predicted genes (Figure 13b).

Figure 13b



4.4 Enrichment analysis

This tab displays and allows the download of the results of the enrichment analysis. After the gene prediction step is completed, DGLinker performs a gene set enrichment analysis (https://en.wikipedia.org/wiki/Gene_set_enrichment_analysis) using Enricher (https://maayanlab.cloud/Enrichr/) and 9 selected databases, with 3 gene sets: the job input set of known genes, the predicted genes and the union of these 2 sets (Figure 14).

Figure 14


For each gene set, the graphs display the 10 most significantly enriched (according to the adjusted p-value) terms of the 9 databases. Both the graphs and the complete results of each individual gene set and database can be downloaded from this tab (Figure 15).


4.5 Download results

This tab allows the user to download the whole set of results as a zip archive (Figure 12).

Figure 15




5. Results files

All results can be downloaded from the webserver as a zip archive named “zip$jobID.zip”. By double-clicking on the zip archive, you should be able to extract its content as most operative systems have compression utilities to deal with zip files preinstalled (Figure 13). However, in the event that you do not have such utilities preinstalled, you can download one from the web (e.g WinRar https://www.win-rar.com). Please find below a description of the content of the results zip archive.

zip$jobID
        |-enrichr$jobID #This folder contains the results of the gene-set enrichment analysis
        |        |-$jobID_$geneSet_$db.txt #raw results of the gene-set analysis
        |        |-$jobID_$geneSet_$db.png #graph of the top 10 hits from the gene-set analysis
        |
        |-graph_dir$jobID
        |        |-$geneName.txt #individual gene interaction file in text format
        |        |-graph_$geneName.html #"standalone" individual gene network in html format
        |
        |-model_results
                |-graph_file$jobID.txt #network interaction file in text format of the knowledge-graph
                |-job_summary_$jobID.txt #job summary file
                |-results_$jobID.txt #model output: gene list and scores
